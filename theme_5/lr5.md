# Задание: Разработка wireframe для информационной системы на основе анализа предметной области

## Цель задания  
Освоить методику проектирования пользовательского интерфейса на ранних этапах разработки ИС путём создания wireframe, соответствующих функциональным требованиям, выявленным при анализе предметной области.

---

## Описание задачи  
На основе примера предметной области **«Фитнес-центр»** из лекции, выполните полный цикл проектирования wireframe в соответствии с методикой, описанной в лекции:

1. **Инвентаризация контента**  
   - Составьте каталог контента в виде таблицы (можно в текстовом виде): перечислите все элементы интерфейса, необходимые для поддержки ролей (клиент, тренер, администратор) и сценариев использования (регистрация, подача заявки, планирование тренировок, отслеживание прогресса и т.д.).  
   - Укажите, к каким экранам/страницам относится каждый элемент.

2. **Визуальная иерархия**  
   - Для каждого ключевого экрана (минимум 3: например, личный кабинет клиента, личный кабинет тренера, панель администратора) определите:
     - Первичные элементы (основные действия, ключевая информация)
     - Вторичные элементы (вспомогательные данные, навигация)
     - Третичные элементы (дополнительные детали)

3. **Wireframe контента**  
   - Создайте черновые каркасы экранов (low-fidelity wireframes) с использованием сетки и блоков. Покажите размещение основных зон: шапка, навигация, основной контент, футер и т.д.

4. **Детальный wireframe**  
   - Доработайте wireframe до среднего уровня детализации (mid-fidelity): добавьте подписи, кнопки, иконки, формы ввода, таблицы — без применения цвета, шрифтов или изображений.  
   - Используйте графические заполнители («lorem ipsum», прямоугольники вместо изображений и т.п.).

5. **Простейший прототип**  
   - Обеспечьте базовую навигацию между экранами (например, переход из списка заявок к деталям заявки, из плана к экрану выполнения упражнений).  
   - Если используете инструмент с поддержкой интерактивности (например, **Visual Paradigm**, **Figma**, **Balsamiq**, **Draw.io**), сделайте кликабельные связи. Если нет - укажите стрелками или подписями логику переходов.

---

## Требования к оформлению

- Работа выполняется **индивидуально**, но по единому сценарию для всех — **«Система управления фитнес-центром»** (см. описание в лекции).  
- Все материалы оформляются в **единый PDF-документ**, включающий:
  - Каталог контента (в виде таблицы)
  - Схему визуальной иерархии (можно в виде текстового списка по экранам)
  - Изображения всех wireframe (минимум 5 экранов: вход/регистрация, личный кабинет клиента, личный кабинет тренера, панель администратора, экран выполнения упражнений)
  - Краткое описание логики переходов (если прототип не интерактивный)
- Работа **не публикуется на GitHub**. Итоговый PDF-файл отправляется в LMS или преподавателю напрямую в соответствии с установленным форматом сдачи.
